  <action>
1. Install MDX and supporting packages:
   ```
   cd tokemon-site && npm install @next/mdx @mdx-js/loader @mdx-js/react @types/mdx @tailwindcss/typography remark-gfm rehype-pretty-code rehype-slug rehype-autolink-headings gray-matter
   ```

2. Delete `next.config.ts` and create `next.config.mjs` with MDX configuration:
   ```javascript
   import createMDX from '@next/mdx'

   /** @type {import('next').NextConfig} */
   const nextConfig = {
     pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
   }

   const withMDX = createMDX({
     options: {
       remarkPlugins: [
         'remark-gfm',
       ],
       rehypePlugins: [
         'rehype-slug',
         ['rehype-pretty-code', { theme: 'one-dark-pro' }],
         ['rehype-autolink-headings', { behavior: 'wrap' }],
       ],
     },
   })

   export default withMDX(nextConfig)
   ```
   Use string-based plugin references (not function imports) for Turbopack compatibility per research pitfall #6.

3. Create `mdx-components.tsx` at the project root (same level as `src/`). This is MANDATORY for @next/mdx to work:
   ```typescript
   import type { MDXComponents } from 'mdx/types'

   export function useMDXComponents(): MDXComponents {
     return {}
   }
   ```

4. Update `globals.css` to add the Tailwind Typography plugin using v4 CSS syntax:
   ```css
   @import "tailwindcss";
   @plugin "@tailwindcss/typography";

   @theme inline {
     --font-sans: var(--font-geist-sans);
     --font-mono: var(--font-geist-mono);
   }
   ```
   IMPORTANT: Use `@plugin` directive (Tailwind v4 syntax), NOT `require()` in a JS config file.

5. Update `tsconfig.json` to include MDX files: add `"**/*.mdx"` to the `include` array so TypeScript recognizes MDX imports. Also add `"./content/**/*.mdx"` to ensure content directory MDX files are included.

6. Verify the build succeeds: `cd tokemon-site && npm run build` (should complete without errors).
